API authentication

Configure CYBER_OIDC_ISSUER, CYBER_OIDC_AUDIENCE, and CYBER_OIDC_JWKS_URL for RS256/ES256 bearer tokens. Map IdP groups to roles; use CYBER_OIDC_DEFAULT_ORG_ID if tokens omit org_id. With OIDC on, CYBER_AUTH_DEV_JWT_ALONGSIDE_OIDC=false disables HS256 dev JWT in production.

Authenticated scans (authenticated_passive)

  • Set profile credential_ref to a vault key; resolve secrets only on the worker via CYBER_VAULT_JSON, CYBER_VAULT_FILE, or HashiCorp KV (VAULT_ADDR + VAULT_TOKEN).
  • Bearer token in vault (username/password empty): passive checks run with Authorization: Bearer … — no Playwright.
  • Password login: set CYBER_PLAYWRIGHT_ADAPTER=1, install Chromium (pip install -e ".[phase2]" + playwright install chromium), and profile options playwright_login_url + selectors. The login URL must be allowlisted.
  • Screenshots go under CYBER_EVIDENCE_DIR as file:… evidence on findings.
